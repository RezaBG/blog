Setting Up and Troubleshooting `code .` in VS Code

Introduction

The `code .` command in Visual Studio Code is a powerful tool that allows you to open the current directory directly from your terminal. This guide will help you set it up and troubleshoot common issues.

Setting Up `code .` Command

To enable the `code .` command, you need to create a symbolic link from the VS Code executable to a directory in your system's `PATH`: 

ln -s "/Applications/Visual Studio Code.app/Contents/Resources/app/bin/code" /usr/local/bin/code

This command makes the `code` command globally accessible from any terminal window.

  • Verify PATH: Ensure `/usr/local/bin` is in your `PATH`: 
    • echo $PATH

Sourcing `.zshrc` and `.zprofile`

If changes to your shell configuration are not taking effect, you may need to source your `.zshrc` or `.zprofile` files: 

source ~/.zshrc
source ~/.zprofile

This ensures that your environment variables, including `PATH`, are correctly updated.
